1. PROJECT OVERVIEW
================================================================================

PURPOSE
-------
This project analyzes tourist arrivals in the Everest region (Namche Bazar area)
of Nepal, spanning from 1995 to 2025. It provides:
  - Data cleaning and validation
  - Exploratory data analysis
  - Statistical summaries and visualizations
  - Outlier and crisis-event analysis (2015 Earthquake, COVID-19)
  - Monthly forecasts for 2026
  - Interactive Streamlit dashboard

DOMAIN
------
Tourism, hospitality, and regional economic planning for the Everest/Sagarmatha
region of Nepal.

5. ALGORITHMS, METHODS & LOGIC
================================================================================

5.1 DATA CLEANING (scripts/data_prep.py)
----------------------------------------

Step 1: Load Raw Data
  - Read entry_data.csv with pandas read_csv()
  - Validate file existence before loading

Step 2: Type Conversion
  - Visit_Date: string → datetime via pd.to_datetime()
  - Enables date arithmetic and extraction

Step 3: Derived Time Features
  - Day = Visit_Date.dt.day
  - Day_of_Week = Visit_Date.dt.day_name()
  - Quarter = Visit_Date.dt.quarter
  - Month_Name = Visit_Date.dt.month_name()

Step 4: Categorical Optimization
  - Convert Nationality, Purpose_of_Visit, Gender, Day_of_Week, Month_Name
    to pandas 'category' dtype
  - Reduces memory usage for repeated string values

Step 5: Data Validation (checks only, no mutation)
  - Missing values: df.isnull().sum()
  - Duplicates: df.duplicated() and Visitor_ID.duplicated()
  - Logical ranges:
      Age: flag if < 10 or > 80
      Expedition_Duration_Days: flag if < 1 or > 90
      Permit_Fee_USD: flag if <= 0

Step 6: Age Grouping
  - pd.cut() with bins [0, 25, 35, 45, 55, 65, 100]
  - Labels: 18-25, 26-35, 36-45, 46-55, 56-65, 65+


5.2 AGGREGATION LOGIC (scripts/data_prep.py)
--------------------------------------------

Yearly Summary (build_yearly_summary):
  - groupby("Year")
  - Aggregations: count(Visitor_ID), sum(Permit_Fee_USD),
    mean(Expedition_Duration_Days), mean(Age)
  - Output columns: Year, Total_Visitors, Total_Revenue_USD, Avg_Duration, Avg_Age

Country Summary (build_country_summary):
  - groupby("Nationality")
  - Same aggregations as yearly
  - Sort by Total_Visitors descending
  - Add Percentage = (Total_Visitors / sum) * 100


5.3 FORECASTING LOGIC (app.py – Streamlit Dashboard)
----------------------------------------------------

Crisis-Year Exclusion:
  - Exclude years 2015 (Nepal Earthquake), 2020, 2021 (COVID-19)
  - Rationale: these years distort seasonal patterns

Monthly Total Forecast for 2026:
  - For each month (1–12):
      Forecast = mean(visitors in that month across all non-crisis years)
  - Simple historical average, no trend adjustment

Segment-wise Forecast:
  - For each segment (Gender, Nationality, Age_Group, Purpose_of_Visit):
      1. Compute share of each category within each month in normal years
         e.g. Male_share_Jan = Male_count_Jan / Total_Jan
      2. For 2026, apply: Forecast_Male_Jan = Total_Forecast_Jan * Male_share_Jan
  - Result: forecasted visitor count per segment per month


5.4 OUTLIER DETECTION (notebook 2)
----------------------------------

IQR (Interquartile Range) Method:
  - Q1 = 25th percentile of yearly visitors
  - Q3 = 75th percentile
  - IQR = Q3 - Q1
  - Lower bound = Q1 - 1.5 * IQR
  - Upper bound = Q3 + 1.5 * IQR
  - Outliers = years where Total_Visitors < lower or > upper

Event-Based Analysis:
  - 2015: Compare visitors to pre-earthquake average
  - 2020–2021: Compare to pre-COVID average
  - Recovery: Compare 2022+ to normal average


5.5 SEASONAL PATTERN (M-SHAPE)
------------------------------
  - Tourism in the Everest region shows two peaks:
      Spring: April–May (pre-monsoon)
      Autumn: October–November (post-monsoon)
  - Low seasons: monsoon (June–Aug), winter (Dec–Feb)
  - Visualized via monthly average bar chart and heatmap
